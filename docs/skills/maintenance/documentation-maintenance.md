# Skill: Documentation Maintenance
> Last updated: 2026-06-28

## Use when

- Adding, moving, merging, archiving, or updating Markdown docs.

## Read first

- [../../11-documentation-maintenance.md](../../11-documentation-maintenance.md)
- [../../00-start-here.md](../../00-start-here.md)

## Steps

1. Decide whether the content belongs in a canonical doc, context doc, skill, or archive.
2. Avoid duplicating existing docs; link to canonical material.
3. Add or update `> Last updated: YYYY-MM-DD` directly below the H1.
4. Update any internal links.
5. Update indexes when adding docs or skills.

## Checks

- Check every Markdown file has a `Last updated` line.
- Check internal links after moves.

## Do not

- Do not leave stale root docs.
- Do not present unverified guesses as current facts.

## Update docs

- Update [../../11-documentation-maintenance.md](../../11-documentation-maintenance.md) when doc conventions change.
- Update the Last updated date of any document changed.
